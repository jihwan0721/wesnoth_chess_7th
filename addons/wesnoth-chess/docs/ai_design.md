# Wesnoth Chess – AI Design

본 문서는 본 프로젝트에서 구현한 체스 기반 AI의 설계 철학, 탐색 방식, 휴리스틱 평가 방법을 설명한다.

---

# 1. 체스 AI 설계 목표

1) Wesnoth 엔진 위에서 동작하는 단순·확장 가능한 체스 AI  
2) 1~2 ply 기반 의사 미니맥스 구조  
3) 가치 기반 평가함수 + 위협도 + 킹 안전도 적용 가능  
4) 난이도(Elo)에 따른 탐색·평가 가중치 조절

---

# 2. 탐색 알고리즘 (search.lua)

## 2.1 1-ply Search
모든 기물의 모든 합법적 이동을 생성하고,  
각 이동 결과를 평가함.

시간 복잡도: O(b)  
(b = branching factor ≈ 12)

## 2.2 2-ply Search
상대가 가장 좋은 응수를 한다는 가정하에 최소 점수의 최댓값을 선택.

시간 복잡도: O(b^2) ≈ 144

---

# 3. 평가 함수 (eval.lua)

각 유닛의 가중치는 전통적 체스 엔진 가중치를 참고하였다.

- Pawn = 100
- Knight = 320
- Bishop = 330
- Rook = 500
- Queen = 900
- King = 20000

총 점수 = 아군 기물 합 – 상대 기물 합

추가적으로 King Safety, Mobility도 도입 가능.

# 5. 오프닝 데이터 (data/openings/)

AI는 라운드2~3에서 간단한 오프닝 파일을 참조하여  
초기 공격 경로 및 타겟팅 우선순위를 조정할 수 있다.
